# ChargerLogAnalysis - 充电日志分析系统方案设计

> 最后更新：2026-05-20  
> 环境检查发现 Drogon 依赖链在 MinGW 下安装成本过高，调整为 Flask (Python) 做 API 层，C++ 核心计算保留。

## 1. 项目背景与目标

构建一个安卓充电日志分析系统，核心功能是从日志中提取 `healthd` 关键字数据进行解析、分析、可视化。

- **核心计算**：C++17（编译为独立可执行文件，Python 通过 subprocess 调用）
- **Web API**：Flask (Python)，环境已有 Python3 + Flask 3.1.3
- **前端**：前后端分离，Vue 3 + TypeScript + Vite + ECharts
- **输入**：CLI 命令行 + Web 页面上传，两者调用同一个 C++ 可执行文件
- **扩展性**：解析器使用类继承设计，未来可适配不同的充电日志格式
- **存储**：SQLite（轻量、单文件、零运维）
- **构建**：CMake（C++ 部分）
- **仓库**：Monorepo

## 4. C++ Core 库设计

### 4.1 解析器体系 (可扩展的类继承)

```cpp
// base_parser.h - 所有日志解析器的基类
class BaseParser {
public:
    virtual ~BaseParser() = default;
    // 判断本解析器能否处理该行
    virtual bool canParse(const std::string& line) const = 0;
    // 解析一行，返回 DataPoint 或 nullopt
    virtual std::optional<DataPoint> parseLine(const std::string& line) const = 0;
    // 解析器名称标识
    virtual std::string parserName() const = 0;
};

// healthd_parser.h - Android healthd 日志解析
class HealthdParser : public BaseParser { ... };

// parser_factory.h - 持有多个解析器，自动匹配
class ParserFactory {
public:
    void registerParser(std::unique_ptr<BaseParser> parser);
    std::optional<DataPoint> parseLine(const std::string& line) const;
    // 新增解析器只需注册，无需改核心逻辑
};
```

### 4.2 Healthd 日志格式

标准 Android healthd 行示例：
```
healthd: battery l=85 v=4150 t=32.5 h=2 st=2 c=450 chg=
```

字段映射：

| 字段 | 含义 | 单位 | 映射 |
|------|------|------|------|
| `l` | 电池电量 | % | `battery_level` (0-100) |
| `v` | 电池电压 | mV | `voltage_mv` |
| `t` | 电池温度 | °C | `temperature_c` (原始值为十分之一度) |
| `st` | 充电状态 | enum | `charging_status` (1=unknown, 2=charging, 3=discharging, 4=not charging, 5=full) |
| `c` | 充电计数 | mAh | `charge_counter` |
| `chg` | 充电电流 | mA | `current_ma` |

### 4.3 数据模型

```cpp
enum class ChargingStatus { Unknown, Charging, Discharging, NotCharging, Full };

struct DataPoint {
    int64_t timestamp = 0;        // 相对毫秒或 Unix 时间戳
    int battery_level = 0;        // 0-100%
    double voltage_mv = 0.0;      // mV
    double temperature_c = 0.0;   // °C
    int current_ma = 0;           // mA
    int charge_counter = 0;       // mAh
    ChargingStatus status = ChargingStatus::Unknown;
};

struct ChargingSession {
    std::string id;
    std::string source_file;
    int64_t start_time = 0;
    int64_t end_time = 0;
    std::vector<DataPoint> data_points;
};

struct SessionStats {
    double avg_charging_rate_pct_per_min;  // 平均充电速率 %/min
    double max_temperature_c;
    double min_voltage_mv, max_voltage_mv;
    double charging_duration_sec;
    int start_level, end_level;
};
```

### 4.4 分析引擎

```cpp
class ChargingAnalyzer {
public:
    // 从数据点中提取充电会话 (过滤放电状态)
    std::vector<ChargingSession> extractSessions(
        const std::vector<DataPoint>& points) const;
    
    // 计算会话统计
    SessionStats computeStats(const ChargingSession& session) const;
    
    // 降采样 (大数据量 → 适合前端图表渲染的适量数据点)
    std::vector<DataPoint> downsample(
        const std::vector<DataPoint>& points, int target_count) const;
};
```

### 4.5 CLI 接口 (Python 通过 subprocess 调用)

```bash
# 解析日志文件，输出 JSON 到 stdout
chargerlog parse input.log --parser healthd

# 解析并导入 SQLite 数据库
chargerlog import input.log --db chargerlog.db

# 列出数据库中所有分析记录
chargerlog list --db chargerlog.db

# 查看某个会话详情 (含降采样后的数据点)
chargerlog show <session_id> --db chargerlog.db --downsample 500

# 输出图表数据 (ECharts series 格式 JSON)
chargerlog chart <session_id> --db chargerlog.db
```

## 5. Flask API 设计

Python 通过子进程调用 C++ 可执行文件，stdin/stdout 传递 JSON 数据。

### cpp_bridge.py

```python
import subprocess
import json

CHARGERLOG_BIN = "chargerlog"  # 或配置路径

def run(*args) -> dict:
    """调用 chargerlog，返回解析后的 JSON"""
    result = subprocess.run(
        [CHARGERLOG_BIN] + list(args),
        capture_output=True, text=True, timeout=60
    )
    if result.returncode != 0:
        raise ChargerLogError(result.stderr)
    return json.loads(result.stdout)
```

### API 路由

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/upload` | 上传日志文件，调用 `chargerlog import`，返回 `{ analysis_id }` |
| GET | `/api/analysis/:id` | 获取分析状态 + 会话统计摘要 |
| GET | `/api/chart/:session_id` | 获取图表数据 (降采样后的 series JSON) |
| GET | `/api/history` | 历史分析列表，支持 `?page=&limit=` 分页 |
| GET | `/api/history/:id` | 单条历史分析详情 |
| DELETE | `/api/history/:id` | 删除分析记录 |

> 进度推送：分析过程通常很快（秒级），先使用轮询（`/api/analysis/:id` 返回 status: processing/done），后续如有大文件需求再加 Flask-SSE。

## 6. SQLite 存储设计

```sql
CREATE TABLE analyses (
    id          TEXT PRIMARY KEY,
    source_file TEXT,
    parser_name TEXT,
    created_at  INTEGER,
    status      TEXT   -- 'processing' | 'done' | 'error'
);

CREATE TABLE sessions (
    id          TEXT PRIMARY KEY,
    analysis_id TEXT REFERENCES analyses(id),
    start_level INTEGER,
    end_level   INTEGER,
    duration_sec REAL,
    max_temp_c  REAL,
    avg_rate    REAL,
    point_count INTEGER
);

CREATE TABLE data_points (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT REFERENCES sessions(id),
    timestamp INTEGER,
    level     INTEGER,
    voltage   REAL,
    temp      REAL,
    current   REAL,
    status    INTEGER
);
```
