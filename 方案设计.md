# ChargerLogAnalysis - 充电日志分析系统方案设计

> 最后更新：2026-05-22  
> 新增阈值分析功能（C++ calcThreshold + CLI --threshold + 前端折线图半透明/加粗渲染）。  
> 环境检查发现 Drogon 依赖链在 MinGW 下安装成本过高，调整为 Flask (Python) 做 API 层，C++ 核心计算保留。

## 1. 项目背景与目标

构建一个安卓充电日志分析系统，核心功能是从日志中提取 `healthd` 关键字数据进行解析、分析、可视化。

- **核心计算**：C++17（编译为独立可执行文件，Python 通过 subprocess 调用）
- **Web API**：Flask (Python)，环境已有 Python3 + Flask 3.1.3
- **前端**：前后端分离，Vue 3 + TypeScript + Vite + ECharts
- **输入**：CLI 命令行 + Web 页面上传，两者调用同一个 C++ 可执行文件
- **扩展性**：解析器使用类继承设计，未来可适配不同的充电日志格式
- **存储**：SQLite（轻量、单文件、零运维）
- **构建**：CMake（C++ 部分）
- **仓库**：Monorepo

## 4. C++ Core 库设计

### 4.1 解析器体系 (可扩展的类继承)

```cpp
// base_parser.h - 所有日志解析器的基类
class BaseParser {
public:
    virtual ~BaseParser() = default;
    // 判断本解析器能否处理该行
    virtual bool canParse(const std::string& line) const = 0;
    // 解析一行，返回 DataPoint 或 nullopt
    virtual std::optional<DataPoint> parseLine(const std::string& line) const = 0;
    // 解析器名称标识
    virtual std::string parserName() const = 0;
};

// healthd_parser.h - Android healthd 日志解析
class HealthdParser : public BaseParser { ... };

// parser_factory.h - 持有多个解析器，自动匹配
class ParserFactory {
public:
    void registerParser(std::unique_ptr<BaseParser> parser);
    std::optional<DataPoint> parseLine(const std::string& line) const;
    // 新增解析器只需注册，无需改核心逻辑
};
```

### 4.2 Healthd 日志格式

实际 Android healthd 行示例：
```
healthd: battery l=100 v=4099 t=31.2 h=2 st=3 c=-889500 fc=2946000 cc=5 chg=
```

字段映射：

| 字段 | 含义 | 单位 | 映射 |
|------|------|------|------|
| `l` | 电池电量 | % | `battery_level_pct` |
| `v` | 电池电压 | mV | `battery_voltage_mv` |
| `t` | 电池温度 | °C | `battery_temperature_c` |
| `c` | 电池电流 | µA | `battery_current_ma` (÷1000 转 mA) |
| `h` | 电池健康 | enum | (未解析: 2=正常) |
| `st` | 充电状态 | enum | (未解析: 2=充电, 3=放电, 5=满) |
| `fc` | 满电容量 | — | (未解析) |
| `cc` | 充电循环次数 | count | `charge_cycle_count` |
| `chg` | 充电器类型 | char | (未解析: 空/a=AC) |

> **canParse 过滤**: 仅匹配同时包含 `healthd` 和 `battery` 的行，排除 ramoops 等文件中的非电池日志（如 `healthd peak:100, real_cap:80...`）。

### 4.3 数据模型

```cpp
struct ChargerDataPoint {
    int64_t elapsed_ms = 0;            // 相对毫秒 (当天 0 点起)
    std::string timestamp_str;         // 原始时间戳字符串 (MM-DD HH:MM:SS.mmm)

    double battery_current_ma = NAN;       // 电池电流 (mA, 由 c 字段 µA÷1000)
    double battery_voltage_mv = NAN;       // 电池电压 (mV)
    double bus_voltage_mv = NAN;           // VBUS 线上电压 (mV)
    double bus_current_ma = NAN;           // 线上电流 (mA)
    double battery_temperature_c = NAN;    // 电池温度 (°C)
    double battery_level_pct = NAN;        // 电池电量百分比 (%)
    double charge_cycle_count = NAN;       // 充电循环次数 (cc)

    bool has(const std::string& field) const;
    double get(const std::string& field) const;
    std::vector<std::string> availableFields() const;
};

struct ThresholdSegment {
```

### 4.5 CLI 接口 (Python 通过 subprocess 调用)

```bash
# 解析日志文件，输出 JSON 到 stdout
chargerlog parse input.log --parser healthd

# 解析并导入 SQLite 数据库
chargerlog import input.log --db chargerlog.db

# 列出数据库中所有分析记录
chargerlog list --db chargerlog.db

# 查看某个会话详情 (含降采样后的数据点)
chargerlog show <session_id> --db chargerlog.db --downsample 500

# 输出图表数据 (ECharts series 格式 JSON)
chargerlog chart <session_id> --db chargerlog.db

# 阈值分析 (可多次指定，在全量数据上计算，不受降采样影响)
chargerlog --json --threshold battery_voltage_mv=4200 --threshold battery_temperature_c=40 <日志目录>
# JSON 输出含 "thresholds" 数组，每项含 field/value/total_time_ms/above_time_ms/above_pct
```

## 5. Flask API 设计

Python 通过子进程调用 C++ 可执行文件，stdin/stdout 传递 JSON 数据。

### cpp_bridge.py

```python
import subprocess
import json

CHARGERLOG_BIN = "chargerlog"  # 或配置路径

def run(*args) -> dict:
    """调用 chargerlog，返回解析后的 JSON"""
    result = subprocess.run(
        [CHARGERLOG_BIN] + list(args),
        capture_output=True, text=True, timeout=60
    )
    if result.returncode != 0:
        raise ChargerLogError(result.stderr)
    return json.loads(result.stdout)
```

### API 路由

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/upload` | 上传日志文件，调用 `chargerlog import`，返回 `{ analysis_id }` |
| GET | `/api/analysis/:id` | 获取分析状态 + 会话统计摘要 |
| GET | `/api/chart/:session_id` | 获取图表数据 (降采样后的 series JSON) |
| GET | `/api/history` | 历史分析列表，支持 `?page=&limit=` 分页 |
| GET | `/api/history/:id` | 单条历史分析详情 |
| DELETE | `/api/history/:id` | 删除分析记录 |

> 进度推送：分析过程通常很快（秒级），先使用轮询（`/api/analysis/:id` 返回 status: processing/done），后续如有大文件需求再加 Flask-SSE。

### Dashboard 图表

| # | 图表 | 类型 | 说明 |
|---|------|------|------|
| 1 | 电压 + 温度 | 双 Y 轴组合图 | 左轴 mV / 右轴 °C，Legend 可切换可见性 |
| 2 | 电流 | 独立折线图 | mA，有正负方向 |
| 3 | 电量 | 独立折线图 | %，Y 轴 0~100 |
| 4 | 剩余循环次数 | 独立折线图 | `额定次数 - cc`，额定值可在图表右上角编辑 |

- 图表卡片支持**拖拽排序**（HTML5 Drag & Drop），排序后 ECharts 实例自动跟随 DOM 移动
- ECharts 原生 `init()`/`dispose()` 管理生命周期，禁用内置 ResizeObserver（避免 minimize 弹回 bug）
- 每个图表支持缩放、tooltip、时间轴联动
- **阈值分析**：每个图表可设置一个阈值，低于阈值的折线段以 ~25% 不透明度**半透明**渲染，≥阈值的折线段**加粗**（width: 4）渲染。前端通过 `threshold.ts` 纯函数从降采样点计算超阈值时间及占比，与 C++ `calcThreshold()` 使用相同的线性插值算法
- 阈值输入框使用 `:value` + `@input` 手动绑定，清空时写入 `null` 关闭阈值效果（避免 `v-model.number` 把空字符串变为 `0` 的 bug）
